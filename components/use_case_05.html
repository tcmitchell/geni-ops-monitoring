Detail of proposed monitoring system components for use case 5

Use case description: Receive notifications when rack, regional, and core control planes are down or when utilization is too high, and visualize or otherwise map the breakdown of control plane traffic by experiment

Use case implementation story: Two types of data are used for this use case. Network reachability checks are simple checks of whether network devices are reachable on their control planes. Maybe each group that wants these checks will run their own from their alerting service (since these are simple ping checks) — here we show the checks being run by an external checker and stored as health check results in a local datastore. The second type of data is time-series metrics counting traffic on network devices. Each rack and each core network will collect these metrics for themselves, and store them in their own local datastore. Then each group that wants to send notifications about network health will run an aggregator that polls the local datastores for any networks or racks they want to monitor, and the aggregator will share recent values with an alerting service, which will detect out-of-bounds values at that group's preference and send alerts.
