Detail of proposed monitoring system components for use case 4

Use case description: Run remote reachability checks of GENI network-listening services including GENI AMs and important non-AM services, and receive notifications when these checks fail

Use case implementation story: A dedicated test process runs remote reachability checks of GENI network-listening services. This process needs access to GENI experimenter credentials in order to run AM API checks. The process is associated with a local datastore, using which it makes check results available as health check metrics. Any group of operators that wants to send notifications on these statistics runs an aggregator, which polls the datastore for any AMs that group wants to send notifications about. The aggregator shares current values with a alerting service, which sends alerts.
